# Repository Skills

Repository-local skills may be added here when sdkwork-utils needs reusable agent or operator workflows that are narrower than the root SDKWork specs.

A real skill must live at `.sdkwork/skills/<skill-name>/SKILL.md`, use lowercase kebab-case, cite the relevant files under `../sdkwork-specs/`, and document its inputs, touched files, commands, and verification.

Do not store generated SDK output, application runtime code, secrets, local credentials, runtime databases, cache, logs, or user-private files in skills.
